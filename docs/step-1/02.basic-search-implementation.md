# 02. 기본 검색: LIKE 기반 동적 쿼리 구현

## 학습 전제조건

이 문서를 학습하기 전에 다음 내용을 알고 있으면 도움이 됩니다:
- Java 기초 문법 (클래스, 인터페이스, 상속)
- 데이터베이스 기초 (SELECT, WHERE, LIKE 연산자)
- HTTP 기초 (GET 요청, 쿼리 파라미터)

---

## 1. 개요

데이터베이스에 적재된 도서 데이터를 검색하는 백엔드 로직을 단계별로 구현해 봅니다.

**이 문서에서 배울 내용:**
- 검색 기능을 왜 별도로 구현해야 하는지
- DTO가 무엇이고 왜 필요한지
- Repository에서 어떻게 검색 쿼리를 만드는지
- QueryDSL을 사용하여 동적 검색을 구현하는 방법

> **중요: 왜 검색이 중요할까요?**
> 도서관에 책이 100만 권 있다고 상상해 보세요. 원하는 책을 찾으려면 제목, 저자, 출판사 등으로 검색할 수 있어야 합니다. 이 검색 기능이 없다면 책을 한 권씩 전부 다 뒤져야 할 것입니다.

---

## 2. 핵심 개념 이해

### 2.1. 검색이란 무엇인가요?

**검색(Search)**은 원하는 데이터를 빠르게 찾는 과정입니다.

도서 검색 예시:
- **제목으로 검색**: "자바"가 들어간 책 찾기
- **저자로 검색**: "남궁성" 쓴 책 찾기
- **ISBN으로 검색**: 고유번호로 정확한 책 찾기

### 2.2. DTO (Data Transfer Object)란?

**DTO**는 계층(Layer) 간에 데이터를 전달하기 위한 객체입니다.

**비유로 이해하기:**
```
식당에서 주문할 때를 생각해 보세요.

[손님] → [주문서] → [주방]

주문서에는:
- 메뉴 이름
- 수량
- 주문 시간

이 주문서가 바로 DTO입니다!
손님과 주방이 서로 다른 언어를 써도,
주문서라는 표준 형식을 통해 소통할 수 있죠.
```

**왜 DTO를 사용할까요?**

1. **보안**: 엔티티(Entity) 내부 정보를 노출하지 않습니다
2. **독립성**: 데이터베이스 구조가 바껴도 API는 그대로 유지할 수 있습니다
3. **명확성**: 전달하려는 데이터가 무엇인지 명확합니다

**요청과 응답 DTO 분리:**
- **Request DTO**: 사용자가 보내는 데이터 (검색어, 페이지 번호 등)
- **Response DTO**: 서버가 응답하는 데이터 (검색 결과, 총 개수 등)

### 2.3. Repository 패턴이란?

**Repository**는 데이터베이스와 직접 소통하는 객체입니다.

**비유로 이해하기:**
```
도서관 사서(Repository)가 책을 찾아주는 상황을 상상해 보세요.

[손님] → [사서] → [책장]

손님은 사서에게 "자바 책 찾아주세요"라고 말합니다.
사서는 책장에서 자바 책을 찾아서 가져다 줍니다.

손님은 책장이 어디에 있는지, 어떻게 찾는지 알 필요가 없습니다.
그냥 사서에게 요청만 하면 됩니다!

이것이 Repository 패턴의 핵심입니다.
```

### 2.4. LIKE 검색이란?

**LIKE**는 SQL에서 부분 일치 검색을 하는 연산자입니다.

**예시:**
```sql
-- 제목에 "자바"가 들어간 책 찾기
SELECT * FROM books
WHERE title LIKE '%자바%'

-- 결과:
- "Java의 정석"
- "자바 코딩의 법칙"
- "이것이 자바다"
```

**LIKE 검색의 특징:**
- 구현이 쉽습니다
- 간단한 검색에는 충분합니다
- 데이터가 많으면 느려집니다
- 색인(Index)를 활용하기 어렵습니다

### 2.5. QueryDSL이란?

**QueryDSL**은 자바 코드로 안전하게 SQL 쿼리를 만들 수 있게 해주는 라이브러리입니다.

**왜 QueryDSL을 사용할까요?**

**일반 SQL의 문제점:**
```sql
-- 문자열로 쿼리 작성 (오타 위험!)
String query = "SELECT * FROM books WHERE title LIKE '%" + keyword + "%'";
// 에러: "book" → "bok" (오타를 컴파일 타임에 알 수 없음)
```

**QueryDSL의 장점:**
```java
// 자바 코드로 쿼리 작성 (타입 세이프!)
queryFactory.selectFrom(book)
    .where(title.contains(keyword))  // 오타가 있으면 컴파일 에러!
    .fetch();
```

**비유로 이해하기:**
```
SQL은 "종이에 손글씨로 쿼리 작성"하는 것과 같습니다.
오타가 나도 실행해봐야 알 수 있죠.

QueryDSL은 "레고 블록 조립"하는 것과 같습니다.
잘못된 블록을 끼우려 하면 컴퓨터가 바로 알려줍니다.
```

## 3. 구현 가이드 (Step-by-Step)

### STEP 1: DTO 정의하기

가장 먼저 계층 간 데이터 전달을 위한 DTO를 만듭니다.

**BookSearchRequest.java (요청 DTO)**

사용자가 검색할 때 보내는 데이터입니다.

```java
package com.nhnacademy.library.core.book.dto;

import com.nhnacademy.library.core.book.domain.SearchType;
import jakarta.validation.constraints.Size;

/**
 * 도서 검색 요청 DTO
 *
 * @record 불변 객체(Immutable Object)로 선언하여 데이터 안전성 확보
 */
public record BookSearchRequest(

    // 검색 키워드 (최대 100자)
    @Size(max = 100, message = "검색어는 100자 이하여야 합니다")
    String keyword,

    // ISBN 번호 (정확한 일치 검색)
    @Size(max = 20, message = "ISBN은 20자 이하여야 합니다")
    String isbn,

    // 검색 유형 (KEYWORD, VECTOR, HYBRID 등)
    SearchType searchType,

    // 벡터 검색을 위한 임베딩 값
    float[] vector,

    // 캐시 워밍업 플래그
    Boolean isWarmUp
) {
    /**
     * 기본 생성자
     * null 값 안전하게 처리하기 위한 메서드 제공
     */
    public String keyword() {
        return keyword != null ? keyword : "";
    }

    public String isbn() {
        return isbn != null ? isbn : "";
    }
}
```

**BookSearchResponse.java (응답 DTO)**

검색 결과를 사용자에게 보여줄 데이터입니다.

```java
package com.nhnacademy.library.core.book.dto;

import com.querydsl.core.annotations.QueryProjection;
import lombok.Getter;
import lombok.NoArgsConstructor;

/**
 * 도서 검색 응답 DTO
 *
 * @QueryProjection QueryDSL의 Projections 기능을 사용하여
 *                  엔티티가 아닌 DTO로 직접 데이터 조회
 */
@Getter
@NoArgsConstructor
public class BookSearchResponse {

    // 도서 ID
    private Long id;

    // ISBN (고유번호)
    private String isbn;

    // 도서 제목
    private String title;

    // 저자명
    private String authorName;

    // 출판사명
    private String publisherName;

    // 도서 내용 미리보기
    private String bookContent;

    // 카테고리
    private String category;

    // 표지 이미지 URL
    private String imageUrl;

    /**
     * QueryProjection을 위한 생성자
     * QueryDSL이 이 생성자를 호출하여 DTO 객체를 생성합니다.
     */
    @QueryProjection
    public BookSearchResponse(
        Long id,
        String isbn,
        String title,
        String authorName,
        String publisherName,
        String bookContent,
        String category,
        String imageUrl
    ) {
        this.id = id;
        this.isbn = isbn;
        this.title = title;
        this.authorName = authorName;
        this.publisherName = publisherName;
        this.bookContent = bookContent;
        this.category = category;
        this.imageUrl = imageUrl;
    }
}
```

> **📝 학습 포인트: DTO vs Entity**
>
> **Entity (엔티티)**: 데이터베이스 테이블과 1:1 매칭되는 클래스
> - DB 구조를 그대로 반영
> - 내부 구현을 노출할 수 있음
>
> **DTO**: 계층 간 데이터 전달용 클래스
> - 필요한 데이터만 선택적으로 포함
> - DB 구조 변경에 영향받지 않음
> - API 명세를 명확히 정의

### STEP 3: QueryDSL 구현체 작성하기

이제 실제 검색 로직을 구현합니다.

**BookRepositoryImpl.java**

```java
package com.nhnacademy.library.core.book.repository;

import com.nhnacademy.library.core.book.domain.QBook;
import com.nhnacademy.library.core.book.dto.BookSearchRequest;
import com.nhnacademy.library.core.book.dto.BookSearchResponse;
import com.nhnacademy.library.core.book.dto.QBookSearchResponse;
import com.querydsl.core.BooleanBuilder;
import com.querydsl.jpa.impl.JPAQueryFactory;
import jakarta.persistence.EntityManager;
import lombok.RequiredArgsConstructor;
import org.apache.commons.lang3.StringUtils;
import org.springframework.data.domain.Page;
import org.springframework.data.domain.PageImpl;
import org.springframework.data.domain.Pageable;
import org.springframework.stereotype.Component;

/**
 * 커스텀 도서 Repository 구현체
 *
 * QueryDSL을 사용하여 동적 검색 쿼리를 생성합니다.
 */
@Component
@RequiredArgsConstructor
public class BookRepositoryImpl implements BookRepositoryCustom {

    private final JPAQueryFactory queryFactory;

    // Q타입: QueryDSL이 생성한 엔티티 메타모델
    QBook book = QBook.book;

    @Override
    public Page<BookSearchResponse> search(Pageable pageable, BookSearchRequest request) {

        // 1. 데이터 조회
        List<BookSearchResponse> results = queryFactory
                .select(new QBookSearchResponse(  // DTO로 직접 프로젝션
                        book.id,
                        book.isbn,
                        book.title,
                        book.authorName,
                        book.publisherName,
                        book.bookContent,
                        book.category,
                        book.imageUrl
                ))
                .from(book)                          // FROM books
                .where(commonWhere(request))         // WHERE 동적 조건 적용
                .offset(pageable.getOffset())        // OFFSET 시작 위치
                .limit(pageable.getPageSize())       // LIMIT 가져올 개수
                .fetch();                            // 실행 및 결과 반환

        // 2. 전체 개수 조회
        long total = queryFactory
                .select(book.count())                // SELECT COUNT(*)
                .from(book)
                .where(commonWhere(request))
                .fetchOne();                         // 단일 결과 반환

        // 3. Page 객체로 래핑
        return new PageImpl<>(results, pageable, total);
    }

    /**
     * 동적 검색 조건 생성
     *
     * BooleanBuilder를 사용하여 null이거나 빈 값인 조건은 자동으로 제외합니다.
     *
     * @param request 검색 요청
     * @return 동적 조건 빌더
     */
    private BooleanBuilder commonWhere(BookSearchRequest request) {
        BooleanBuilder builder = new BooleanBuilder();

        // 키워드 검색 (제목 또는 저자명)
        if (StringUtils.isNotEmpty(request.keyword())) {
            builder.and(
                book.title.containsIgnoreCase(request.keyword())     // 제목 포함 (대소문자 무시)
                    .or(book.authorName.containsIgnoreCase(request.keyword()))  // 또는 저자명 포함
            );
        }

        // ISBN 정확히 일치 검색
        if (StringUtils.isNotEmpty(request.isbn())) {
            builder.and(book.isbn.eq(request.isbn()));  // isbn = '입력값'
        }

        return builder;
    }
}
```

### STEP 4: 동적 쿼리 이해하기

**BooleanBuilder**는 상황에 따라 쿼리가 변하는 "동적 쿼리"를 만들 때 사용합니다.

**예시: 쇼핑몰 필터**

사용자가 다양한 조건으로 검색할 수 있습니다:
- 카테고리만 선택
- 가격대만 선택
- 카테고리 + 가격대 + 브랜드 모두 선택

**BooleanBuilder 사용 예시:**

```java
BooleanBuilder builder = new BooleanBuilder();

// 사용자가 입력한 조건만 추가
if (category != null) {
    builder.and(product.category.eq(category));
}

if (minPrice != null) {
    builder.and(product.price.goe(minPrice));  // goe = greater or equal
}

if (brand != null) {
    builder.and(product.brand.eq(brand));
}

// 결과: 조건에 따라 다른 쿼리 생성!
// category만 입력: WHERE category = 'books'
// category + price: WHERE category = 'books' AND price >= 10000
// 모두 입력: WHERE category = 'books' AND price >= 10000 AND brand = 'A'
```

## 6. 학습 체크리스트

다음 내용을 이해했는지 확인해 보세요:

- [ ] DTO가 무엇이고 왜 필요한지 설명할 수 있다
- [ ] Repository 패턴의 역할을 이해한다
- [ ] LIKE 검색의 기본 원리를 안다
- [ ] QueryDSL을 사용하는 이유를 설명할 수 있다
- [ ] BooleanBuilder로 동적 쿼리를 만들 수 있다
- [ ] Pageable로 페이징 처리를 할 수 있다
